# 美术指导文档：谐振视界 (Resonance Horizon)

**项目名称：** Project Harmony
**视觉风格：** 科幻神学 / 极简几何 / 故障艺术
**核心隐喻：** 战斗即是调频，世界即是乐谱。

---

## 1. 场景与环境设计 (Environment Design)

游戏设定在“音乐的内在逻辑”之中，因此场景核心是**“可视化的声学空间”**，而非传统的物理环境。

### 1.1. 地面：量化网格 (The Quantized Grid)

- **基础材质：** 深邃的黑色或深蓝色虚空，表面带有微弱的反射性，如同黑曜石镜面。
- **动态网格：** 地面上铺设着发光的网格线，象征五线谱或音序器的量化格子。
- **节拍响应：** 网格线的亮度与发光强度会随着背景音乐的BPM（每分钟节拍数）进行脉冲式闪烁。
- **玩家交互：** 当玩家移动时，其脚下的网格会产生类似水波纹的顶点位移（Vertex Displacement）效果，仿佛玩家是行走在张力表面上的一个波源。

### 1.2. 障碍物：固化静默 (Solidified Silence)

- **概念：** 阻挡玩家移动的并非实体岩石，而是“无效的数据块”或“被冻结的声波波形”。
- **造型：** 形态为高耸的黑色玄武岩状柱体，其表面带有均衡器（EQ）频谱起伏的动态视觉效果。
- **受击反馈：** 当子弹击中障碍物时，它不会碎裂，而是会短暂地“亮起”并发出低沉的共振声，仿佛被瞬间激活。

### 1.3. 边界：事件视界 (Event Horizon)

- **地图边缘：** 游戏世界的边界并非传统的“空气墙”，而是一道**“白噪音屏障”**。
- **视觉效果：** 边界由不断跳动的黑白噪点构成一堵视觉墙。当玩家靠近时，游戏画面会受到干扰（例如轻微的扭曲），以警告玩家正在接近无序的混沌领域。

## 2. 界面与交互设计 (Diegetic UI/UX)

UI应作为游戏世界的一部分（Diegetic UI）存在，感觉像是玩家核心系统的有机组成部分，而非简单地贴在屏幕表面的图层。

### 2.1. 核心HUD：合成器面板

- **血条 (谐振完整度)：**
    - **位置：** 位于玩家核心正下方，或屏幕正下方，呈现为弧形。
    - **样式：** 一条正弦波。
    - **状态反馈：**
        - **满血：** 波形平滑、稳定，发出金色光芒。
        - **低血：** 波形变得锯齿化（趋向方波），颜色转为红色，频率变得不稳定。

- **弹药/冷却 (序列器状态)：**
    - **样式：** 围绕玩家核心旋转的环形刻度。
    - **状态反馈：**
        - **自动施法点：** 亮起的光点，随着节拍节奏扫过圆环。
        - **手动施法就绪：** 对应的快捷键图标高亮，并伴有微弱的电流特效。

### 2.2. 伤害数字：频率反馈

- **暴击/完美节拍：** 伤害数字显示为金色的波纹扩散效果，数字本身带有**故障艺术（Glitch）**效果，仿佛过高的伤害导致了显示器的渲染错误。
- **普通伤害：** 采用极简的白色像素字体，数字快速上浮并消散。
- **不和谐伤害 (自我伤害)：** 数字呈现紫色，并带有向下流淌的视觉效果，如同液体泄漏。

### 2.3. 拾取物：数据包

- **经验值 (XP)：** 表现为漂浮的音符符号或微小的正四面体，而非传统的发光球体。被玩家吸收时，它们会化作一条光线，最终汇入玩家核心。
- **宝箱/升级：** 一个悬浮的、被锁住的几何谜题盒子（类似魔方）。打开时，它会解体并重组成代表新升级的图标。

## 3. 特效规范 (VFX Guidelines)

特效是游戏表现力的灵魂，必须严格区分“和谐”与“噪音”两种对立的视觉语言。

### 3.1. 玩家技能 (和谐之力)

- **关键词：** 流畅、连贯、透明、发光。
- **形状语言：** 圆形、流线型、对称几何体。
- **色彩叠加：** 使用 Additive（线性减淡）混合模式，创造明亮通透的效果。
- **消失方式：** 优雅地缩小直至变成一个光点消失，或像水面涟漪一样逐渐淡出。

### 3.2. 敌人攻击 (噪音污染)

- **关键词：** 尖锐、破碎、阻塞、不透明。
- **形状语言：** 三角形、锯齿、碎玻璃、不对称形状。
- **色彩叠加：** 使用 Multiply（正片叠底）或不透明材质，产生遮挡和压迫感。
- **消失方式：** 瞬间破碎成像素块，或像老式电视关机一样闪烁并消失。

### 3.3. 听感疲劳视觉化 (The Fatigue Filter)

这是游戏最关键的视觉反馈机制，通过全局后处理（Post-Processing）实现，其强度与“听感疲劳指数”（Aesthetic Fatigue Index, AFI）直接挂钩。

| AFI 范围 | 状态 | 视觉效果 |
| :--- | :--- | :--- |
| **< 0.3** | 清澈 | 开启 **Bloom (泛光)**，世界清晰锐利，色彩饱和。 |
| **0.3 - 0.6** | 浑浊 | 开启 **Film Grain (胶片噪点)**，画面边缘光晕开始轻微抖动。 |
| **> 0.8** | 过载 | 开启 **Chromatic Aberration (色差)** 和 **Scanlines (扫描线)**，除红色和高亮白色外，其他颜色饱和度大幅降低，世界变得灰暗。 |

## 4. 动画原则 (Animation Principles)

### 4.1. 玩家动画：插值 (Interpolation)

- **核心感觉：** 玩家的移动和旋转应极其平滑，带有缓入缓出（Ease-in/Ease-out）曲线，仿佛悬浮在无摩擦的磁场中。

### 4.2. 敌人动画：量化 (Quantization)

- **核心感觉：** 为了体现“不和谐”与“机械感”，敌人的动画可以采用较低的帧率（如每秒12帧）或瞬移式的步进插值（Step Interpolation）。这种视觉上的“卡顿感”将与玩家的丝滑流畅形成鲜明对比，制造出令人不安的威胁感。

## 5. 独立开发者技术落地指南 (Indie Implementation Guide)

考虑到开发成本，本方案极度依赖 Shader（着色器）而非传统的手工建模和贴图。

### 5.1. 推荐技术栈

- **引擎：** Unity (URP) 或 Godot 4。
- **核心Shader技术：**
    - **Fresnel Effect (菲涅尔效应):** 用于所有角色和子弹，使物体中心透明、边缘发光，产生“能量体”的质感。
    - **Vertex Displacement (顶点位移):** 用于让背景网格和敌人随着音乐BPM跳动。
    - **Kawase Blur / Bloom:** 高质量的泛光是营造“霓虹科幻”感的关键。

### 5.2. 资源制作捷径

- **材质：** 避免手绘贴图。所有物体使用纯色材质，并赋予强烈的PBR（物理渲染）属性，如高光滑度和高自发光强度。
- **几何体生成：** 使用代码或简单的Blender修改器（如Ico Sphere, Torus Knots）生成复杂的几何形状。
- **敌人模型：** 对于复杂的敌人，可以使用一团被力场束缚的粒子来表现，从而省去蒙皮和骨骼动画的制作成本。

### 5.3. 性能优化 (Survivor-like 专项)

- **GPU Instancing:** 鉴于同屏可能出现成百上千的几何体子弹，必须使用GPU Instancing技术进行渲染。
- **对象池 (Object Pooling):** 实施严格的对象池管理策略，以避免因垃圾回收（GC）导致的卡顿（即“丢帧”），这在音乐节奏游戏中是致命的。
- **VFX Graph (Unity) / GPUParticles3D (Godot):** 使用基于GPU的粒子系统来处理大量的背景和特效。

## 6. 概念图参考描述 (用于AI生成或人工想象)

- **画面中心：** 一个璀璨的、由三道旋转金环构成的正十二面体（代表玩家），正在向四周发射出如同五线谱般的激光束。
- **画面四周：** 在黑暗的虚空中，无数红色的、锯齿状的碎片（代表敌人）正在向中心逼近。
- **光影与特效：** 激光击中碎片时，会炸裂出蓝色的数据代码块。
- **全局滤镜：** 画面边缘带有微弱的红蓝3D眼镜效果（色差），暗示整个系统正处于轻微的过载状态。
